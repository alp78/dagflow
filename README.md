# Dagflow

Dagflow is a financial data operations platform for SEC reference data and 13F shareholder holdings.
It captures regulated source data, loads it into a shared PostgreSQL workspace, transforms it with
dbt, lets reviewers correct inputs with immediate cross-dataset recalculation, and exports immutable
delivery bundles once review is complete.

## Architecture

```
Source files (EDGAR)
  -> Capture & land (CSV + Parquet + manifest)
  -> Load to raw Postgres tables
  -> dbt transform (flat models, no staging/intermediate)
  -> Publish review snapshot
  -> Human review & editing (React UI)
  -> Approve & export (CSV bundles + audit parquet)
```

The platform is workflow-driven. Each workflow is a self-contained package under `workflows/` with
its own Dagster assets, dbt models, source adapters, and topology metadata. The `config.json` file
at the repo root registers all active workflows.

## Unified Pipeline

The `security_shareholder` workflow runs as a single pipeline with one review/approve/export cycle
covering both datasets:

- **security_master** — SEC issuer identity, outstanding shares, free-float, investability
- **shareholder_holdings** — 13F filer positions, portfolio weights, derived pricing

Sources: `sec_company_tickers`, `sec_company_facts`, `holdings_13f`, `holdings_13f_filers`

## Dagster Asset Graph

The asset graph is introspected from actual Dagster definitions and rendered as an interactive
DAG in the UI. The topology is generated by `ops/scripts/generate_dag_topology.py` using the
Graphviz `dot` engine for zero-crossing layout.

```
Capture Tickers  -> Raw Tickers  -+
Capture Facts    -> Raw Facts    -+-> Securities -> Review Securities -> Securities Export -+
                                                                                           +-> Write CSVs
Capture Holdings -> Raw Holdings -+                                                        |
Capture Filers   -> Raw Filers   -+-> Holdings   -> Review Holdings   -> Holdings Export  -+
```

Each node maps to a Dagster asset key. Nodes sharing the same op (e.g., all capture sub-steps)
are aligned to the same column. The UI shows real-time step status with breathing animation for
running steps and interactive tooltips with asset metadata, file paths, and timing.

Two Dagster jobs:
- `security_shareholder_job` — capture through review (main pipeline)
- `security_shareholder_export_job` — dbt export + CSV writing (triggered by review validation sensor)

## End-to-End Flow

1. **Get Data** — capture source files from EDGAR for a business date
2. **Load Data** — launch the Dagster pipeline: capture -> load -> dbt transform -> publish review
3. **Review** — inspect and edit the review workbench (securities table with expandable holdings)
4. **Approve & Export** — validate both datasets, trigger export job via sensor
5. **Reset** — clear the workspace to load another date

## Storage Model

### Filesystem

- **Source landing**: raw CSVs, canonical Parquet, manifests per source/date
- **Export bundles**: immutable delivery artifacts per dataset/date/run

### PostgreSQL

| Schema | Purpose |
|---|---|
| `control` | Pipeline registry, state, source file registry, calendar days |
| `raw` | Typed raw contract tables (loaded from source files) |
| `marts` | dbt-transformed mart tables (securities, holdings) |
| `review` | Editable review tables for the active workspace |
| `export` | dbt export tables (approved rows shaped for delivery) |
| `audit` | Change log, workflow events, review data issues |
| `observability` | Pipeline runs, step runs, failures, quality results |
| `workflow` | Review state machine, publish/validate/export functions |

## Review Workbench

The React UI provides a unified master-detail view:

- **Securities table** — main review grid with expandable holdings per security
- **Inline cell editing** — click to edit, Enter commits immediately, blur stages for batch commit
- **Recalculation cascade** — editing shares_outstanding recomputes free_float, investability,
  and all linked holdings percentages
- **DAG flowchart** — real-time pipeline status with per-asset breathing animation and tooltips
- **Calendar picker** — business day calendar with source availability indicators
- **Issues panel** — data quality issues with click-to-navigate
- **Metadata panel** — run ID, status, timing

### Editable columns

| Dataset | Column | Recalculates |
|---|---|---|
| security_master | `shares_outstanding_raw` | free_float_pct, investability, free_float_shares, investable_shares, materiality, linked holdings % |
| shareholder_holdings | `shares_held_raw` | holding_pct_of_outstanding, derived_price_per_share |
| shareholder_holdings | `reviewed_market_value_raw` | portfolio_weight |

Client-side validation rejects non-numeric input before sending to the API.

## Export Bundles

Each dataset writes an immutable bundle under `<export_root>/<storage_prefix>/<date>/<run_id>/`:

| File | Content |
|---|---|
| `reviewed.csv` | Final approved rows — the main deliverable |
| `baseline.csv` | Pre-review raw data for comparison |
| `audit_events.parquet` | Row/column-level audit trail |
| `manifest.json` | Bundle metadata, checksums, counts, source lineage |

## DAG Topology Generation

The UI DAG is generated from Dagster definitions, not hardcoded:

```bash
docker compose exec dagster-user-code-security-shareholder \
  python /workspace/ops/scripts/generate_dag_topology.py \
  --out /workspace/apps/ui-react/src/generated
```

The script:
- Introspects `AssetsDefinition` objects for keys, dependencies, ops, groups, metadata
- Expands multi-step assets (capture_sources with 4 sub-steps) into individual nodes
- Detects cross-job boundaries and adds synthetic edges
- Uses Graphviz `dot` engine with rank constraints for zero-crossing layout
- Outputs `dag_topology.json` imported by the UI at build time

Run this after any change to asset definitions, dbt model dependencies, or step definitions.

## Local Development

```bash
# Start all services
docker compose up -d

# Rebuild specific services
docker compose up -d --build api
docker compose up -d --build dagster-user-code-security-shareholder

# Apply a migration
cat db/migrations/043_fix_export_column_names.sql | docker compose exec -T postgres psql -U postgres -d dagflow

# Regenerate DAG topology
docker compose exec dagster-user-code-security-shareholder \
  python /workspace/ops/scripts/generate_dag_topology.py \
  --out /workspace/apps/ui-react/src/generated

# Type-check UI
cd apps/ui-react && npx tsc --noEmit

# Verify API
curl -s http://localhost:8000/pipelines | python3 -m json.tool
```
